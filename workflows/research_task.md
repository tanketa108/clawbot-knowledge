# Workflow — Research Task

Use this workflow to execute an item from `research_backlog.md`.

## Steps

1. Select one task and state the decision it informs.
2. Gather primary sources first when available.
3. Extract facts separately from interpretation.
4. Add source-backed evidence to `evidence_log.md`.
5. Update `questions.md` only if the question is answered, refined or superseded.
6. Update `thesis.md` only if the conclusion changes materially.
7. Update sector/framework files if the insight is reusable across companies.
8. Record material changes in `changelog.md`.

## Output standard

Every completed research task should leave behind:

- one concise written conclusion,
- source references,
- evidence-log update,
- explicit thesis impact: supports / challenges / neutral,
- remaining uncertainty.
