# Workflow — Quarterly Update

## Steps

1. Store official source documents in `sources/`.
2. Create `updates/YYYY-QX-results.md` from template.
3. Extract key metrics and segment performance.
4. Compare results to existing thesis and open questions.
5. Add discrete items to `evidence_log.md`.
6. Update `questions.md`.
7. Modify `thesis.md` only if thesis impact is material.
8. Record changes in `changelog.md`.
9. Update `indexes/recent_updates.md`.

## Rule

Quarterly updates should not silently rewrite the thesis. Material thesis changes require changelog entries.
